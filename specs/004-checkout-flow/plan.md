## Summary

Implement a 4-step, session-only checkout flow at `/checkout/*` вЂ” cart review, customer info form, order summary, and order confirmation вЂ” using React Context + `sessionStorage` (mirroring the existing `CartContext` pattern). No payment gateway, no authentication, no database. Orders are generated client-side and held in sessionStorage for the confirmation display only.

## Architecture Decisions

### 1. CheckoutContext mirrors CartContext

`context/CheckoutContext.tsx` uses the same `useReducer` + `sessionStorage` pattern as `context/CartContext.tsx`. State shape:

```
CheckoutState {
  customerInfo: CustomerInfo | null
  completedOrder: Order | null
}
```

Actions: `SET_CUSTOMER_INFO`, `SET_COMPLETED_ORDER`, `CLEAR_CHECKOUT`

Scoped to `app/checkout/layout.tsx` вЂ” not the root layout (cart state is global, checkout state is checkout-scoped).

**Storage key**: `lumiere_checkout` (object with both `customerInfo` and `completedOrder`)

### 2. Step Guard Pattern

Each guarded page uses `useEffect` + `router.replace()` and renders `null` during the check:

```
/checkout/info:         guard в†’ cart empty?          в†’ /checkout/cart
/checkout/review:       guard в†’ cart empty?          в†’ /checkout/cart
                                customerInfo null?   в†’ /checkout/info
/checkout/confirmation: guard в†’ completedOrder null? в†’ /checkout/cart
```

### 3. Validation вЂ” Pure Functions in `lib/checkout.ts`

`validateCustomerInfo(info)` returns `ValidationErrors` (empty object = valid). Exported as a pure function for direct Vitest testing. No library dependency.

Phone validation: strip `\D`, test `/^(92)?0?3[0-9]{9}$/` вЂ” covers local (03XX), and international (+923XX) formats.

### 4. Order Number Generation

`generateOrderNumber()` in `lib/checkout.ts`:

```
LP-${new Date().getFullYear()}-${String(Date.now() % 99999).padStart(5, '0')}
```

Example: `LP-2026-04731`. Unique within a single sequential checkout session.

### 5. Checkout Layout вЂ” Stripped-Down Header

`app/checkout/layout.tsx` renders:
- Minimal header: LumiГЁre Parfums wordmark (links to `/`) вЂ” no main nav, no CartBadge
- `<CheckoutProgress />` below header
- `{children}` below progress bar

This is intentional вЂ” checkout pages suppress the main site navigation to reduce distraction and keep users focused on completing the order (standard e-commerce conversion UX).

### 6. `<CheckoutProgress />` Uses `usePathname()`

Active step is derived from `usePathname()` вЂ” no props needed. Step-to-path mapping:

```
'cart'         в†’ /checkout/cart
'info'         в†’ /checkout/info
'review'       в†’ /checkout/review
'confirmation' в†’ /checkout/confirmation
```

### 7. Order Placement Flow

On "Place Order" click in `<OrderReview />`:
1. Call `placeOrder(cartItems)` from `useCheckoutContext()` вЂ” creates `Order` object
2. `CheckoutContext` dispatches `SET_COMPLETED_ORDER`
3. `CartContext` dispatches `CLEAR_CART` (cart cleared immediately)
4. `router.push('/checkout/confirmation')`

`placeOrder` is defined in `CheckoutContext` (accesses both contexts) вЂ” it generates the order number, builds `OrderLineItem[]` snapshots, and computes `grandTotal`.

### 8. Reuse `CartContext` вЂ” No Duplication

Cart quantity/remove mutations on `/checkout/cart` use the **existing** `CartContext` dispatch (`ADD_ITEM` for quantity changes, new `REMOVE_ITEM` action). The cart review page is a new view of the same cart state вЂ” not a separate copy.

**New CartAction needed**: `REMOVE_ITEM: { type: 'REMOVE_ITEM'; payload: { productId: string } }` вЂ” added to `types/cart.ts` and `cartReducer`.
